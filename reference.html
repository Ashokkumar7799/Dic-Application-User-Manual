Quick Reference & Troubleshooting

A condensed reference covering the full workflow, recommended parameter values, common problems with their solutions, and a glossary of key terms.

Complete Workflow Summary

Follow these steps in order every time you perform a DIC test. Skipping steps may result in physically meaningless results or failed analysis.

StepModuleMinimum Required ActionsGo/No-Go Check
⭐ License & Startup Paste license key → Tick Terms checkbox → Click Activate Green banner visible. Continue button enabled.
1 Image Acquisition Import image files or capture via camera → Click Execute Module "✓ Images Ready!" banner visible.
2 Calibration Choose method (Auto Grid / Import / Manual Scale) → Apply or complete calibration Calibration status shows confirmed mm/px ratio or "✓ Calibration Successful".
3 Focus Quality Click RUN ANALYSIS → Review rating and heatmap Rating must be EXCELLENT or ACCEPTABLE. Do not proceed with MARGINAL or POOR.
4 Pattern Validation Upload speckle image → Click Analyze Pattern → Check results Coverage 25–50%. Quality rating = Optimal.
5 DIC Analysis & Results Load sequence → Draw ROI → Set Subset Size → Click Run DIC Analysis → Browse results → Export CSV Analysis completes without errors. Results graphs show expected trend.

Key Parameter Guidelines

These values represent best-practice defaults. Adjust based on your specific camera, optics, and test conditions.

ParameterWhere SetRecommended ValueRationale
Speckle CoveragePattern Validation25–50%Below 25%: too few features for tracking. Above 50%: individual dots merge, making detection unreliable.
Mean Speckle DiameterPattern Validation (result)3–7 pxSpeckles smaller than 2 px are sub-pixel and unreliable. Larger than 10 px wastes area without improving tracking.
Subset SizeDIC Analysis3–5× mean speckle diameterMust contain enough pattern-unique texture for the correlation to be unambiguous. Too small: noise. Too large: loses spatial resolution.
Grid StepDIC Analysis0.5–1× Subset SizeSmaller than the subset is acceptable for dense fields. Much larger than the subset wastes spatial resolution.
Blur ThresholdFocus Quality20 (FLIR class), 1900 (FLIR hardware)FLIR industrial cameras produce very high Laplacian-variance scores. Standard USB cameras score much lower. The Live Focus tool sets this automatically when you click Start Camera.
Calibration Images (Auto Grid)Calibration15–20 imagesMinimum 10 required. More images from more varied angles improve calibration accuracy and reduce reprojection error.
Save Rate (Live Camera)AcquisitionMatch to test speedFor quasi-static tests (mm/min crosshead speed): 1–2 FPS. For impact or dynamic tests: maximum available (15 FPS).
Skip FramesAcquisition (Preview)0 for quasi-static; N for high-speedFor high-speed sequences (>100 frames), use skip to reduce the dataset to a manageable size for analysis.

Troubleshooting

ProblemLikely CauseSolution
"Invalid license key" errorKey does not match this Machine ID, or key was pasted with extra whitespaceRe-copy the key from your vendor email. Paste into a plain text editor first to check for extra characters. Then paste into the application.
Activate button stays greyed outTerms and Conditions checkbox is not tickedScroll through the Terms text area and tick the "I have read and agree…" checkbox.
License works but application doesn't startMissing Python dependencies or corrupted installationContact your vendor with the error message that appears in any popup or console window.
Focus quality rating is POORCamera is physically out of focusUse the Live Focus Evaluator (Module 3 → Live Focus button) to adjust the camera focus ring while monitoring the real-time focus score.
Heatmap is uniformly cold (blue)Global blur — entire image is defocusedRefocus the camera from scratch. If using a fixed-lens setup, check that the working distance matches the lens specification.
Calibration fails with detection errorPattern not detected in one or more imagesCheck: image sharpness, correct pattern type selected, correct square/dot size entered, target fully visible in each image with no cut-off corners.
RMS reprojection error >1.5 pxInconsistent target images or incorrect dimension inputsRemove images where the target was blurry or the view was oblique. Verify that all physical dimension inputs are in mm and are accurate.
Speckle coverage is <25%Pattern is too sparseApply an additional light spray coat of black paint (without fully re-painting the white base coat). Allow to dry fully before re-acquiring.
Speckle coverage is >50%Over-sprayed or dots have mergedRepaint the specimen — apply a clean white base coat first, then a lighter speckle application.
DIC analysis fails mid-sequenceSubset correlation fails on a frame due to excessive deformation or image quality dropAdaptive reference is always enabled, but a sudden large load step can exceed the search range. Try reducing the subset size or using a coarser grid step. Check that no acquired frames are blurry or overexposed.
All strain values are zero or near-zeroCalibration not completed, or reference frame is the deformed imageVerify calibration is applied. Confirm that Image 1 in the sequence is the undeformed reference state.
CSV export produces an empty fileNo results data available (analysis not run or stopped before any frames completed)Re-run the analysis. Ensure at least one frame was processed before attempting export.

Glossary of Key Terms

TermDefinition
DICDigital Image Correlation — an optical measurement technique that tracks the motion of image features (speckles) to compute full-field surface displacement and strain.
Reference FrameThe first image in the sequence, representing the undeformed state of the specimen. All displacements are measured relative to this image.
SubsetA small square image patch used as the correlation template for tracking a single point. Also called a facet.
ROIRegion of Interest — the portion of the image that is analyzed. Restricting the ROI to the specimen surface improves speed and accuracy.
Strain FieldA spatial map showing the strain value at every tracked point across the specimen surface.
Principal StrainsMajor and Minor strains — the maximum and minimum normal strains at a point, found by rotating to the principal coordinate system. Useful for identifying failure-critical locations.
Speckle PatternA random, high-contrast dot pattern applied to the specimen surface. Provides the unique local image texture required for DIC tracking.
Coverage %The fraction of the image area (or ROI) covered by speckle dots. Ideal range: 25–50%.
RMSERoot Mean Square Error — used here to quantify image noise. Lower RMSE = lower noise floor = better measurement sensitivity.
Laplacian VarianceThe focus score metric used by the Focus Quality module. Computed as the variance of the second-order spatial derivative of the image. Higher value = sharper image.
CV (Coefficient of Variation)Standard deviation divided by mean, expressed as a percentage. A lower CV indicates more uniform focus across the image.
Machine IDA hardware-derived unique identifier for your computer, used to bind a software license to a specific machine.
mm/pxMillimeters per pixel — the calibration scale factor that converts pixel distances in the image to real-world physical distances.
