Module 5

DIC Analysis & Results

The core analysis module. Tracks the deformation of your specimen across the image sequence using full-field Digital Image Correlation. Computes displacement and strain fields for every tracked point in every frame. The Results section is integrated directly within this module.

Figure 5.1 — DIC Analysis module main screen

5.1   Left Control Panel — DIC Configuration

The left scrollable panel contains all settings needed to configure and run the DIC analysis. Work through the controls from top to bottom before clicking Run.

Images Section

ControlDescription
Load Image Sequence (blue button)Opens a file browser for selecting the image sequence. Select all files in one operation (hold Shift or Ctrl). The first image in the selected set automatically becomes the reference frame. All subsequent images are treated as deformed frames.
Live Capture (purple button)Opens an integrated camera capture panel, allowing you to build the image sequence in real-time from a connected FLIR or USB camera.
Image count status labelDisplays one of: "No images loaded" (default), or "X images loaded" (green) confirming the number of frames ready for analysis.
The reference frame is always the first image in the sequence. All displacement and strain values are computed relative to this reference. Ensure your first image shows the specimen in the undeformed state (zero-load condition).

ROI (Region of Interest) Selection

Define the area of the image that will be tracked. Restricting the ROI to the specimen surface (excluding fixtures, grips, and background) significantly reduces computation time and improves result quality.

ControlDescription
Rectangle (radio button)The ROI will be drawn as a rectangle. Suitable for most specimens.
Polygon (radio button)The ROI will be drawn as a free-form polygon (multiple vertices). Use for irregularly shaped specimens or when the ROI must closely follow a curved edge.
Select ROI (button)Activates drawing mode. Click on the right-panel canvas to define the ROI. For Rectangle: click-drag to draw. For Polygon: click each vertex in sequence, then close by clicking the starting point or pressing Enter.
Undo Pt (orange — Polygon mode only)Removes the most recently placed polygon vertex. Click multiple times to remove multiple points.
Clear (button)Removes the current ROI entirely. The canvas returns to showing the full image. This button is disabled until an ROI has been defined.
ROI info labelDisplays the coordinates and pixel dimensions of the current ROI (e.g., "ROI: (120, 80) — 640×480 px").

Analysis Mode

ModeDescription
Full-Field Grid (radio button)Places a regular grid of tracking points (subsets) evenly spaced across the entire ROI. This produces displacement and strain field maps covering the whole specimen surface. This is the standard DIC mode for most applications.

Subset (Facet) Size

The subset is a small square image patch centered on each tracking point. The DIC algorithm searches for this patch in each deformed image to determine how much it has moved.

ControlDescription
Subset Size slider (range: 10–150 px, default: 51)Sets the side length of the tracking subset in pixels. Larger subsets are more stable and resistant to noise but capture coarser deformation gradients. Smaller subsets capture finer deformation detail but can fail in low-contrast or severely deformed regions.
Direct entry boxType a specific subset size value. Press Enter or click away to apply. The slider updates to match.
Quick Presets: 21 / 31 / 51 / 75 (buttons)Jump immediately to commonly used subset sizes. Equivalent to typing the value manually.
Use the subset size recommendation from the Pattern Validation module as your starting point. The general rule: subset size should be 3–5× the mean speckle diameter. For a mean speckle of 8 px, try a subset size of 25–40 px.

Grid Step (Point Spacing)

ControlDescription
Grid Step slider (range: 8–252 px, default: 90)Sets the center-to-center spacing between adjacent tracking points in the full-field grid. Smaller step = more tracking points = higher spatial resolution but longer computation time. Larger step = fewer points = faster but coarser field maps.
Direct entry boxType a specific grid step value. The slider updates to match.

Strain Calculation

Select the formulation used to compute spatial strains.

ControlDescription
Engineering (ε = λ - 1)Standard engineering strain based on the original undeformed length. Suitable for small deformations.
Green-Lagrange (E)A finite strain measure that is invariant to rigid body rotations. Highly recommended for large deformations and rotations.
True Strain (ln)Logarithmic strain formulation (ε = ln(λ)). Useful for large plastic deformations often used in materials testing.

5.2   Run Analysis Controls

ControlDescription
Run DIC Analysis (large green button)Starts the full DIC correlation engine in a background thread. The right-panel canvas begins updating frame-by-frame as results are computed. This button is disabled once analysis begins.
Stop (red button — visible only during analysis)Halts the running analysis after the current frame completes. Any frames analyzed up to that point are retained and available for visualization and export.
Analysis Progress barFills from left to right as each frame is processed. Shows completion percentage.
"Frame X / Y" labelShows the current frame being analyzed (e.g., "Frame 45 / 120"). Updates in real-time.
Do not close or navigate away from the DIC Analysis module while analysis is running. Doing so will terminate the analysis and the results will not be saved.

5.3   Right Panel — Image Canvas

The right panel displays your reference image in a zoomable canvas. After setting the ROI, the grid of tracking points becomes visible as an overlay.

Figure 5.2 — Reference image canvas showing subset grid and scale bar

Elements visible on the canvas before and during analysis:

  • Green grid overlay — dots or squares marking each tracking point position on the reference image. The spacing matches your Grid Step setting.
  • White scale bar — a calibrated distance marker shown in the corner of the image. Only present if calibration has been completed.
  • ROI boundary — a colored border showing the defined analysis region.

During analysis, the canvas switches to the current deformed frame and overlays a color heatmap showing the selected strain or displacement field in real-time.

Post-Analysis Visualization Controls

After analysis completes, a visualization toolbar appears below the canvas:

ControlDescription
Strain Field dropdownSelect which measurement field to visualize: Strain_X [%], Strain_Y [%], Shear_Strain_XY [%], Major_Strain [%], Minor_Strain [%], Displacement_X [mm], Displacement_Y [mm].
Colormap dropdownColor scheme for the heatmap overlay. Options include: jet, viridis, plasma, hot, coolwarm, turbo, RdBu, and others. Jet is the default.
Opacity slider (0.0–1.0)Controls how transparent the heatmap overlay is relative to the underlying image. Set to 1.0 for full overlay; 0.0 to see only the original image without any overlay.
◀ (Previous frame button)Steps backward one frame in the result sequence. Shows the heatmap for that frame.
"Frame X / Y" result counterShows your current viewing position in the analyzed sequence (e.g., "Frame 12 / 120").
▶ (Next frame button)Steps forward one frame in the result sequence.

Overlays

ControlDescription
Show Point IDs (toggle)Displays the unique integer ID number on top of each tracked point in the visualization canvas.
Scale Bar (toggle)Shows or hides the calibrated physical scale bar in the corner of the exported images.
Frame Number (toggle)Overlays the current frame number (e.g., "Frame 45") directly onto the image.

5.4   Results & Data Export

The Results view is accessed by clicking the Results button (or it appears automatically after analysis). It shows interactive graphs of the computed fields over time and provides export functionality.

Figure 5.3 — DIC Analysis Results panel

Info label (top right)Summarizes the dataset: total analyzed frames, number of tracked points (e.g., "120 frames | 350 tracked points").
Image Order (button)Opens a dialog listing the reference and all deformed images in the order they were analyzed. Useful for traceability and confirming that the correct sequence was used.
Back to ViZ (red button)Returns to the DIC Analysis canvas view so you can continue browsing the strain heatmap overlay frame-by-frame.

Export Buttons

ButtonDescription
📥 Export CSVExports strain and displacement data. In Full Field mode, all tracked points are exported. In Point Mode, only the currently selected points are exported. The format is automatically chosen: Excel (.xlsx) for datasets with ≤16,000 points, or multiple CSV files for larger datasets.
📹 Export VisualizationsSaves every frame as an individual PNG image with the heatmap overlay applied, plus a combined MP4 video of all frames. Overlays (scale bar, frame number) are burned into the exported images based on your current toggle settings.
📈 Export All GraphsExports the time-series line plots (U, V, Strain X, Strain Y, Major Strain, Minor Strain) as individual PNG images into a selected folder.
🔄 Back to DIC AnalysisReturns to the main DIC Configuration and canvas view.

Right Panel — Interactive Graph

The graph plots the currently selected field value on the Y axis, with frame number on the X axis. Each line represents one tracked point in the field of view.

  • Multiple lines — each corresponds to one spatial tracking point. For large datasets, up to 500 points are sampled for display to keep the visualization responsive. The full dataset is always exported.
  • Hover interaction — move the cursor over any line to see a tooltip with the exact value at that frame for that point.
  • Zoom and pan — click and drag on the graph to zoom into a region. Right-click or use the toolbar to reset the zoom.
The graph sampling (max 500 display points) does not affect the exported data. The export always contains every tracked point from every analyzed frame. Use the exported files for rigorous data analysis in external tools (Excel, MATLAB, Python, etc.).

Exporting Data & File Formats

Export CSV — Data Format

The Export CSV button creates a multi-sheet file. Each measurement metric gets its own sheet (or its own CSV file for large datasets). Within each sheet, the data is pivoted: rows are Frames, and columns are individual Points. Three summary columns (Average, Min, Max) are appended to each sheet.

Sheets created (in order):

Sheet / File NameContents
UHorizontal displacement (X-direction) per point per frame
VVertical displacement (Y-direction) per point per frame
Strain_X [%]Normal strain in X per point per frame
Strain_Y [%]Normal strain in Y per point per frame
Major_Strain [%]Maximum principal strain per point per frame
Minor_Strain [%]Minimum principal strain per point per frame
Point_PositionReference X and Y coordinates for each point (2 rows: X, Y)
MagnitudeTotal displacement magnitude per point per frame

Sheet layout example (e.g., sheet Strain_X [%]):

FramePoint_0Point_1Point_2…AverageMinMax
00.0120.0150.009…0.0120.0090.015
10.0340.0410.028…0.0340.0280.041
For datasets with more than 16,000 tracked points, the application automatically switches to multiple CSV files instead of a single Excel workbook (Excel row/column limits). Each metric is saved as a separate CSV file named dic_results_U.csv, dic_results_Strain_X_pct.csv, etc. A _summary.txt file with displacement and strain statistics is also generated alongside the data.

Export Visualizations — Folder Structure

When you click Export Visualizations, you select an output folder. The application creates the following structure:

[Selected Folder]/
  ├── Strain_X_pct_frames/
  │   ├── frame_0000.png
  │   ├── frame_0001.png
  │   └── … (one PNG per analyzed frame)
  └── Strain_X_pct_visualization.mp4

The folder and video names reflect the currently selected strain field (e.g., Major_Strain_pct_frames/). Each exported PNG includes the heatmap overlay, colorbar, and any enabled overlays (scale bar, frame number text).

Export All Graphs — Output

Exports each time-series graph type as a separate PNG image into your chosen folder:

[Selected Folder]/
  ├── Graph_Displacement_X.png
  ├── Graph_Displacement_Y.png
  ├── Graph_Strain_X.png
  ├── Graph_Strain_Y.png
  ├── Graph_Major_Strain.png
  └── Graph_Minor_Strain.png
